OpenPadang dikembangkan dengan pendekatan **bertahap dan berkelanjutan**.
Roadmap berikut merupakan panduan, bukan target dengan deadline mengikat.

## 🎯 Phase 0: Foundation Setup — IN PROGRESS

**Tujuan:** Project online dengan landing page minimal dan struktur dasar
yang dapat dilanjutkan kapan pun.

- ✅ Folder structure dan project documentation
- ✅ Git dan public GitHub repository
- ✅ pnpm monorepo (site/, app/, packages/shared/)
- ✅ Astro Starlight documentation site
- ✅ SvelteKit application skeleton
- ✅ Design tokens, Tailwind 4, Lucide, Inter
- ✅ LICENSE (MIT + CC-BY 4.0) + CONTRIBUTING.md
- 🔲 Vercel deployment

## 📊 Phase 1: Catalog & Documentation

**Tujuan:** Documentation site dengan katalog 20+ sumber data publik
terdokumentasi.

- Pelengkapan Data Catalog dengan minimal 20 sumber publik
- Per source: URL, format, license, refresh rate, contact, reliability rating
- Detail page per source
- Accessibility audit dan Lighthouse mobile audit
- Soft announcement publik

## 🌍 Phase 2: First Curated Dataset

**Tujuan:** Satu dataset production-ready, dapat diakses melalui GitHub raw
URL atau API, dengan dokumentasi lengkap.

**Kandidat starter (lowest hanging fruit):**

- OSM POI Padang (satu query Overpass API → JSON)
- BMKG real-time wrapper
- BPS Padang demographic time-series

## ⚡ Phase 3: Showcase — Padang Real-Time Status PWA

**Tujuan:** PWA yang mengonsumsi data publik (BMKG, PetaBencana, OWM) sebagai
demonstrasi konkret manfaat Open Data.

Fitur yang direncanakan:

- Gempa real-time dan tsunami warning
- Cuaca dan AQI per area
- Banjir crowd-source dari PetaBencana
- Map view dengan Leaflet
- PWA installable dengan offline-cache
- Web Push notification untuk gempa M≥5 di Sumbar

## 🔌 Phase 4: API Gateway

**Tujuan:** REST API publik dengan dokumentasi OpenAPI 3.0.

- SvelteKit `+server.ts` endpoints
- Interactive API docs (Swagger UI / Scalar)
- Rate limiting dan caching
- Code examples (cURL, JavaScript, Python)

## 📈 Phase 5: Visualization & Data Stories

**Tujuan:** Dashboard publik dan data journalism.

**Ide story:**

- "Demografi Padang dalam Visualisasi" (BPS data)
- "Mengapa Lubuk Buaya Rentan Banjir?" (geo + historical)
- "Trans Padang: Koridor dengan Frekuensi Terendah" (apabila data tersedia)

## 🌐 Phase 6: Outreach & Community

**Tujuan:** Project tidak lagi solo — terbentuknya kontributor, partnership,
dan recognition.

- Surat resmi kepada Diskominfo Padang
- Approach UNAND/UNP untuk kolaborasi penelitian
- Submit ke DataPortals.org
- Article di LinkedIn dan Twitter
- Partnership hackathon
