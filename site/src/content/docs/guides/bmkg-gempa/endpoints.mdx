BMKG menyediakan 3 endpoint JSON publik untuk data gempa. Seluruh endpoint
dapat diakses tanpa autentikasi.

## Base URL

```
https://data.bmkg.go.id/DataMKG/TEWS/
```

---

## 1. `autogempa.json` — Gempa Terbaru

<Badge text="Real-time" variant="success" />

**Endpoint:**
```
GET https://data.bmkg.go.id/DataMKG/TEWS/autogempa.json
```

**Refresh rate:** Latency sekitar 6 detik dari kejadian
**Returns:** Satu gempa paling baru

### Response Schema

```json
{
  "Infogempa": {
    "gempa": {
      "Tanggal": "string — '12 Mei 2026'",
      "Jam": "string — '10:15:32 WIB'",
      "DateTime": "string ISO 8601 UTC — '2026-05-12T03:15:32+00:00'",
      "Coordinates": "string — '-0.95,100.36' (lat,lng)",
      "Lintang": "string — '0.95 LS' atau '0.95 LU'",
      "Bujur": "string — '100.36 BT' atau '100.36 BB'",
      "Magnitude": "string — '3.4' (decimal)",
      "Kedalaman": "string — '10 km'",
      "Wilayah": "string — '21 km BaratLaut KOTAPADANG-SUMBAR'",
      "Potensi": "string — 'Tidak berpotensi tsunami' atau peringatan",
      "Dirasakan": "string optional — 'II Padang' (skala MMI)",
      "Shakemap": "string optional — filename peta intensitas"
    }
  }
}
```

### Shakemap Image

Apabila field `Shakemap` tersedia, image dapat diakses pada:
```
https://data.bmkg.go.id/DataMKG/TEWS/{Shakemap}
```

Format: JPEG. Menampilkan peta intensitas guncangan.

## 2. `gempaterkini.json` — 15 Gempa Terakhir

<Badge text="Historical" variant="caution" />

**Endpoint:**
```
GET https://data.bmkg.go.id/DataMKG/TEWS/gempaterkini.json
```

**Refresh rate:** Sekitar 5 menit
**Returns:** Array 15 gempa terbaru (umumnya M≥5 atau notable)

### Response Schema

```json
{
  "Infogempa": {
    "gempa": [
      {
        "Tanggal": "...",
        "Jam": "...",
        "DateTime": "...",
        "Coordinates": "...",
        "Lintang": "...",
        "Bujur": "...",
        "Magnitude": "...",
        "Kedalaman": "...",
        "Wilayah": "...",
        "Potensi": "..."
      }
      // ... 14 entry lainnya
    ]
  }
}
```

Skema per entry sama dengan `autogempa.json`, dengan pengecualian field
`Dirasakan` dan `Shakemap` yang tidak selalu tersedia.

## 3. `gempadirasakan.json` — Gempa Dirasakan Warga

<Badge text="Community" variant="tip" />

**Endpoint:**
```
GET https://data.bmkg.go.id/DataMKG/TEWS/gempadirasakan.json
```

**Refresh rate:** Variabel (tergantung kejadian dan laporan)
**Returns:** Array 15 gempa yang dilaporkan dirasakan oleh masyarakat

### Response Schema

Skema sama dengan `gempaterkini.json`, dengan tambahan field `Dirasakan`
yang berisi skala MMI dan lokasi:

```json
{
  "Dirasakan": "III Padang Sidempuan, II Padang"
}
```

**Interpretasi skala MMI:**

| Skala | Deskripsi |
|---|---|
| I | Tidak terasa (hanya terdeteksi alat) |
| II-III | Terasa di lantai atas bangunan |
| IV-V | Terasa banyak orang, barang gantung bergoyang |
| VI-VII | Plester retak, dinding pecah |
| VIII+ | Kerusakan bangunan |

## 💎 Endpoint Hidden Gems (Tidak Terdokumentasi Resmi)

Selain 3 endpoint utama, BMKG memiliki **endpoint internal di Google Cloud
Storage** yang menyediakan data jauh lebih komprehensif. Endpoint ini
dipakai oleh aplikasi internal InaTEWS namun dapat diakses publik.

### 🌍 `live30event.xml` — 194 Events Real-Time

<Badge text="Tested & Verified" variant="success" />

**Endpoint:**
```
GET https://bmkg-content-inatews.storage.googleapis.com/live30event.xml
```

**Refresh:** Real-time (mendekati 1-2 menit latency)
**Returns:** Hingga ~200 events terbaru (nama "live30" misleading)
**Coverage:** Seluruh Indonesia
**Format:** XML

**Keunggulan vs `gempaterkini.json`:**

| Aspek | `gempaterkini.json` | `live30event.xml` |
|---|---|---|
| Jumlah events | 15 | **~200** (10-12x lebih banyak) |
| Magnitude minimum | M5+ | **M2.0+** |
| Event ID unique | Tidak ada | ✅ `<eventid>` untuk dedup |
| Status (confirmed/prelim) | Tidak ada | ✅ `<status>` |
| Focal mechanism | Tidak ada | ✅ `<fokal>` |
| Time precision | Detik | **Millisecond** |
| Format | JSON | XML |

**Response Schema:**

```xml
<?xml version="1.0"?>
<Infogempa>
 <gempa>
  <eventid>bmg2026jgpg</eventid>
  <status>confirmed</status>
  <waktu>2026/05/12  16:20:05.389</waktu>
  <lintang>-8.35</lintang>
  <bujur>116.16</bujur>
  <dalam>15</dalam>
  <mag>2.6</mag>
  <fokal>undetermined</fokal>
  <area>Sumbawa Region, Indonesia</area>
 </gempa>
 <!-- ~200 entries... -->
</Infogempa>
```

**Field reference:**

| Field | Type | Deskripsi |
|---|---|---|
| `eventid` | string | Unique ID BMKG, format `bmg{YYYY}{xxxx}` |
| `status` | string | `confirmed` atau `preliminary` |
| `waktu` | string | UTC datetime dengan millisecond `YYYY/MM/DD HH:MM:SS.SSS` |
| `lintang` | number | Latitude (negative = LS) |
| `bujur` | number | Longitude (negative = BB) |
| `dalam` | number | Kedalaman km |
| `mag` | number | Magnitude |
| `fokal` | string | Focal mechanism (`undetermined`, `normal`, `thrust`, `strike-slip`) |
| `area` | string | Wilayah dalam Bahasa Inggris |

<Aside type="tip" title="Recommended untuk Disaster Monitoring">
	Untuk app monitoring gempa Padang/Sumbar dengan threshold M2+, endpoint
	ini jauh lebih useful dibanding `gempaterkini.json` (yang cuma M5+).
	Bisa filter realtime per region menggunakan field `area`.
</Aside>

### 🗺️ `indo_faults_lines.geojson` — Patahan/Sesar Indonesia

<Badge text="Visualization Data" variant="caution" />

**Endpoint:**
```
GET https://bmkg-content-inatews.storage.googleapis.com/indo_faults_lines.geojson
```

**Size:** ~310 KB
**Format:** GeoJSON FeatureCollection (LineString)

Data fault lines (garis patahan) Indonesia untuk visualisasi geospasial.
Critical untuk:

- Map app yang tampilin gempa di context fault zone
- Educational visualization tentang seismic risk
- Tsunami risk analysis (gempa dekat fault offshore = risk tinggi)
- Layer reference untuk Padang yang berada di **Sumatera Megathrust zone**

**Response Schema:**

```json
{
  "type": "FeatureCollection",
  "name": "indo_faults_lines",
  "crs": { "type": "name", "properties": { "name": "..." } },
  "features": [
    {
      "type": "Feature",
      "properties": {
        "Primary ID": "",
        "Secondary ID": "rank 1"
      },
      "geometry": {
        "type": "LineString",
        "coordinates": [
          [ 105.203723, -5.328199 ],
          [ 105.292491, -5.466554 ]
        ]
      }
    }
  ]
}
```

**Use case Padang/Sumbar:**

Filter fault lines yang relevan untuk Padang (bbox approximate):

```js
function isPadangArea(feature) {
	const coords = feature.geometry.coordinates;
	return coords.some(([lon, lat]) => 
		lon >= 99 && lon <= 102 && lat >= -3.5 && lat <= 0.5
	);
}

const padangFaults = data.features.filter(isPadangArea);
```

## Endpoint Lainnya (Limited)

| Endpoint | Status |
|---|---|
| `https://repogempa.bmkg.go.id/eventcatalog` | Web UI only — tidak ada public API. Untuk request bulk historical, harus submit form via web. |
| `https://inatews.bmkg.go.id/web/realtime` | Web portal real-time |
| `https://rtsp.bmkg.go.id/` | InaTEWS portal (membutuhkan login) |
| `https://api.bmkg.go.id/publik/gempa-*` | ❌ Tidak ada (hanya cuaca yang punya endpoint `/publik`) |

<Aside type="caution" title="Stabilitas Hidden Endpoints">
	Endpoint di Google Cloud Storage (`bmkg-content-inatews.storage.googleapis.com`)
	tidak terdokumentasi resmi. BMKG dapat mengubah URL atau format tanpa
	notifikasi. Untuk production, **monitor field schema secara berkala**
	dan implement defensive parsing.
</Aside>

## CORS

Seluruh endpoint mendukung CORS (`Access-Control-Allow-Origin: *`), sehingga
dapat dipanggil langsung dari aplikasi browser atau SPA tanpa proxy server.

## Rate Limit

BMKG tidak mempublikasikan rate limit resmi. Berdasarkan observasi komunitas:

- Polling dengan interval kurang dari 1 detik berisiko menghasilkan
  rate-limit atau IP block
- Polling 5-10 detik per request aman, namun kurang efisien
- **Rekomendasi:** polling 30 detik untuk `autogempa.json` dengan
  caching di sisi server

## Autentikasi

Tidak diperlukan. Seluruh endpoint bersifat publik tanpa API key.
