API BMKG cukup sederhana, namun memiliki sejumlah quirks yang perlu
diantisipasi. Daftar berikut adalah hasil observasi dari penggunaan di
production maupun komunitas — penting untuk diketahui sebelum deployment.

## ⚠️ Timezone — Hati-Hati dalam Parsing

Field `DateTime` sudah dalam format ISO 8601 UTC (`...+00:00`), namun
`Tanggal` dan `Jam` ditampilkan dalam WIB (UTC+7). Hindari mencampur kedua
representasi tersebut.

```js
// ❌ SALAH: parse Jam tanpa konteks timezone yang jelas
const time = new Date(`${gempa.Tanggal} ${gempa.Jam}`); // ambigu

// ✅ BENAR: gunakan DateTime yang sudah berformat ISO
const time = new Date(gempa.DateTime); // jelas UTC
// Tampilkan ke pengguna dengan timezone Asia/Jakarta
const display = time.toLocaleString('id-ID', { timeZone: 'Asia/Jakarta' });
```

## ⚠️ Field `Magnitude` Bertipe String

Seluruh field response BMKG bertipe **string**, termasuk nilai numerik.

```js
// ❌ SALAH: perbandingan langsung
if (gempa.Magnitude >= 5) // "5.4" >= 5 → coercion behaviour

// ✅ BENAR: konversi terlebih dahulu
if (parseFloat(gempa.Magnitude) >= 5) // eksplisit
```

## ⚠️ Format Field `Coordinates`

Format: `"lat,lng"` sebagai string (dipisah koma tanpa spasi). Sementara
`Lintang` dan `Bujur` memiliki format display (`"0.95 LS"` atau `"0.95 LU"`).

```js
// Parsing coordinates yang aman
const [lat, lng] = gempa.Coordinates.split(',').map(parseFloat);

// Atau parsing dari Lintang/Bujur (memerlukan logika tambahan)
function parseLintang(str) {
	const [val, dir] = str.split(' ');
	return parseFloat(val) * (dir === 'LS' ? -1 : 1);
}
```

## ⚠️ Field `Dirasakan` Bisa Bernilai `"-"`

Berdasarkan testing per Mei 2026 pada endpoint `autogempa.json`, field
`Dirasakan` **selalu ada di response**, namun bernilai `"-"` (string dash)
apabila belum ada laporan dari warga. Bukan `undefined` atau missing.

`Shakemap` umumnya tersedia untuk gempa yang sudah dianalisis BMKG.

```js
// ❌ SALAH: anggap "-" sebagai data valid
console.log(gempa.Dirasakan); // bisa "-"

// ✅ BENAR: check explicit untuk "-" atau empty
const dirasakan =
	gempa.Dirasakan && gempa.Dirasakan !== '-'
		? gempa.Dirasakan
		: 'Belum ada laporan dari warga';

const shakemapUrl = gempa.Shakemap
	? `https://data.bmkg.go.id/DataMKG/TEWS/${gempa.Shakemap}`
	: null;
```

<Aside type="note">
	Pada endpoint `gempaterkini.json` dan `gempadirasakan.json`, field
	`Dirasakan` dan `Shakemap` dapat benar-benar tidak ada (bukan `"-"`)
	untuk entry tertentu. Tetap gunakan defensive optional checking.
</Aside>

## ⚠️ Format `Wilayah` Tidak Konsisten

Format field `Wilayah` bervariasi dalam hal kapitalisasi dan format compound:

- `"21 km BaratLaut KOTAPADANG-SUMBAR"` (compound, tanpa spasi)
- `"95 km Tenggara MENTAWAI-SUMBAR"` (normal)
- `"Laut Sumatera Barat"` (deskriptif)

Untuk filtering, lakukan normalisasi terlebih dahulu:

```js
function normalizeWilayah(s) {
	return s
		.toUpperCase()
		.replace(/\s+/g, ' ')
		.replace(/-/g, ' ')
		.trim();
}

const isPadang = (g) =>
	normalizeWilayah(g.Wilayah).includes('PADANG');
```

## ⚠️ `gempaterkini.json` Tidak Selalu Berurutan

Meskipun umumnya tersorted descending berdasarkan waktu, terkadang data
correction dari BMKG menyebabkan entry baru muncul di tengah array. Apabila
urutan penting, lakukan sorting eksplisit:

```js
const sorted = data.Infogempa.gempa.sort(
	(a, b) => new Date(b.DateTime) - new Date(a.DateTime)
);
```

## ⚠️ Rate Limiting

BMKG tidak mempublikasikan rate limit resmi. Berdasarkan observasi:

- Polling kurang dari 1 detik berisiko menghasilkan HTTP 429 atau IP block
- Polling 5-10 detik aman namun kurang efisien
- **Rekomendasi production:** polling 30 detik dengan caching di server

**Strategi caching:**

```js
// Server-side cache (Vercel Edge / Cloudflare Workers)
let cache = { data: null, expires: 0 };

export async function GET() {
	if (Date.now() < cache.expires && cache.data) {
		return json(cache.data);
	}

	const res = await fetch('https://data.bmkg.go.id/DataMKG/TEWS/autogempa.json');
	const data = await res.json();

	cache = { data, expires: Date.now() + 30_000 }; // 30 detik
	return json(data);
}
```

## ⚠️ Field `Potensi` Bertekstual Bahasa Indonesia

Field `Potensi` selalu dalam Bahasa Indonesia:
- `"Tidak berpotensi tsunami"` (mayoritas kasus)
- `"Berpotensi tsunami"` — CRITICAL
- `"Berpotensi tsunami untuk wilayah XXX"` — area spesifik

Untuk pemeriksaan programmatik potensi tsunami:

```js
const isTsunamiPotential = (g) =>
	g.Potensi.toLowerCase().includes('berpotensi tsunami') &&
	!g.Potensi.toLowerCase().startsWith('tidak');
```

## ⚠️ Outage dan Maintenance

BMKG terkadang melakukan maintenance (umumnya di luar jam kerja WIB).
Endpoint dapat merespons dengan:
- HTTP 503 (service unavailable)
- HTML error page (bukan JSON)
- Response kosong

**Defensive coding:**

```js
try {
	const res = await fetch('https://data.bmkg.go.id/DataMKG/TEWS/autogempa.json');

	if (!res.ok) {
		throw new Error(`HTTP ${res.status}`);
	}

	const contentType = res.headers.get('content-type');
	if (!contentType?.includes('json')) {
		throw new Error('Unexpected content type');
	}

	const data = await res.json();
	if (!data?.Infogempa?.gempa) {
		throw new Error('Unexpected schema');
	}

	return data.Infogempa.gempa;
} catch (err) {
	console.error('BMKG fetch failed:', err);
	// Fallback: gunakan cache lama atau tampilkan warning ke pengguna
	return null;
}
```

## ⚠️ Tidak Tersedia Webhook atau Push Notification

BMKG tidak menyediakan webhook atau push notification resmi. Mekanisme yang
tersedia hanya polling.

Untuk near-real-time experience:
- Polling 30 detik (acceptable untuk kebanyakan use case)
- Polling 6 detik mengikuti latency BMKG, namun kurang efisien
- Server-sent Events / WebSocket tidak tersedia

Alternatif: gunakan Twitter API untuk monitor akun `@infoBMKG` sebagai trigger.

## Limitations OpenPadang Highlight

| Limitation | Workaround |
|---|---|
| Tidak ada API key atau autentikasi | Tidak diperlukan (kebijakan publik BMKG) |
| Tidak ada webhook | Implementasi polling 30 detik |
| Skema dapat berubah tanpa notifikasi | Defensive parsing, monitoring tanggal verifikasi |
| Coverage hanya Indonesia | Sesuai untuk use case Padang |
| Magnitude rendah mungkin tidak dipublikasikan | Gunakan `gempadirasakan.json` untuk gempa kecil |
| Data historis > 1 bulan terbatas | Gunakan `repogempa.bmkg.go.id` (kurang stabil) |
