Overpass API menggunakan bahasa query khusus bernama **Overpass QL** untuk
mengakses dan memfilter data OSM. Halaman ini memberikan reference sintaks
dengan fokus pada penggunaan untuk Padang.

## Base Endpoint

```
POST https://overpass-api.de/api/interpreter
GET  https://overpass-api.de/api/interpreter?data=<query>
```

**Catatan:** POST direkomendasikan untuk query panjang. Body request adalah
form-encoded `data=<query>`.

### Server Alternatif

Apabila server utama lambat, gunakan mirror:

- `https://overpass.kumi.systems/api/interpreter`
- `https://overpass.openstreetmap.fr/api/interpreter`
- `https://overpass.osm.ch/api/interpreter`

## Struktur Query Dasar

```
[out:FORMAT][timeout:SECONDS];
SCOPE_STATEMENT;
out OUTPUT_TYPE;
```

**Contoh minimal:**

```
[out:json][timeout:25];
node["amenity"="cafe"]({{bbox}});
out body;
```

## Format Output

Ditentukan dengan `[out:FORMAT]`:

| Format | Penjelasan |
|---|---|
| `json` | JSON (paling sering digunakan untuk integrasi) |
| `xml` | XML (default OSM) |
| `csv` | Tabular CSV (memerlukan output specifier) |
| `popup` | HTML untuk display |

## Element Types

OSM memiliki 3 jenis elemen:

| Type | Deskripsi | Contoh |
|---|---|---|
| `node` | Titik tunggal (lat/lng) | POI seperti cafe, ATM |
| `way` | Garis atau area (urutan nodes) | Jalan, building outline |
| `relation` | Kumpulan elements | Rute bus, multipolygon |

```
node["amenity"="cafe"];
way["building"];
relation["type"="route"]["route"="bus"];
```

## Filter berdasarkan Tag

### Equals

```
["amenity"="hospital"]
```

### Exists (tag ada, value apa pun)

```
["amenity"]
```

### Not Equals

```
["amenity"!="hospital"]
```

### Regex

```
["name"~"Padang"]              // contains
["name"~"^Padang"]             // starts with
["amenity"~"^cafe|restaurant$"]  // multiple values
```

### Multiple Filters (AND logic)

```
node["amenity"="restaurant"]["cuisine"="indonesian"]({{bbox}});
```

### OR Logic (Union)

```
(
  node["amenity"="restaurant"];
  node["amenity"="cafe"];
);
```

## Area Filter

Dua cara membatasi area pencarian:

### 1. Bounding Box

```
node["amenity"="cafe"](-1.05, 100.25, -0.85, 100.50);
//                     south  west    north  east
```

### 2. Area Name (Polygon Administratif)

```
area["name"="Kota Padang"]["admin_level"="5"]->.padang;
node["amenity"="cafe"](area.padang);
```

**Admin levels untuk Indonesia:**

| Level | Region |
|---|---|
| 2 | Country (Indonesia) |
| 4 | Province (Sumatera Barat) |
| 5 | Kota/Kabupaten (Kota Padang) |
| 6 | Kecamatan (Padang Barat) |
| 7 | Kelurahan (Belakang Tangsi) |

## Output Statement

Format `out`:

```
out [OUTPUT_TYPE] [LIMIT];
```

**Output types:**

| Type | Returns |
|---|---|
| `body` | Element + tags, no geometry |
| `geom` | Element + tags + geometry (untuk way/relation) |
| `center` | Element + tags + lat/lng titik tengah (untuk way) |
| `bb` | Element + tags + bounding box |
| `count` | Hanya count, no elements |
| `ids` | Hanya IDs |

**Limit:**

```
out body 10;       // max 10 elements
out body 100;
```

## Recursive Operators

Untuk dapat detail elemen yang terhubung (e.g., nodes dari way):

```
(
  way["highway"](area.padang);
  >;  // Recurse down: ambil nodes dari ways
);
out body;
```

## Limit & Performance

### Timeout

Default 180 detik. Untuk query kompleks, tingkatkan:

```
[out:json][timeout:300];
```

### Maxsize

Limit ukuran response (default 512 MB). Jarang perlu diubah.

```
[out:json][maxsize:1073741824]; // 1 GB
```

### Output Limit

Batasi jumlah element yang dikembalikan:

```
out body 1000;
```

## Quick Reference Summary

| Aspek | Value |
|---|---|
