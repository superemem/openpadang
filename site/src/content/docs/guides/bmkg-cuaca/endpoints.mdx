BMKG menyediakan endpoint tunggal untuk prakiraan cuaca dengan parameter
hierarkis sesuai level administratif.

## Base Endpoint

BMKG menyediakan **dua endpoint** untuk prakiraan cuaca:

```
1. PRIMARY (v2 — support semua adm level):
   GET https://cuaca.bmkg.go.id/api/df/v1/forecast/adm

2. LEGACY (hanya adm4):
   GET https://api.bmkg.go.id/publik/prakiraan-cuaca
```

**Rekomendasi:** gunakan endpoint v2 (`cuaca.bmkg.go.id`) untuk fleksibilitas
penuh dari level provinsi hingga kelurahan dalam single request.

Header `Accept: application/json` disarankan disertakan untuk menghindari
redirect ke landing page HTML.

## Query Parameters

Endpoint v2 (`cuaca.bmkg.go.id/api/df/v1/forecast/adm`) menerima parameter level wilayah:

| Parameter | Level | Format | Contoh (Padang) | Returns |
|---|---|---|---|---|
| `adm1` | Provinsi | 2 digit | `13` (Sumatera Barat) | 19 kabupaten/kota |
| `adm2` | Kabupaten/Kota | 5 digit | `13.71` (Kota Padang) | 11 kecamatan |
| `adm3` | Kecamatan | 8 digit | `13.71.09` (Kuranji) | 9 kelurahan |
| `adm4` | Kelurahan/Desa | 13 digit | `13.71.09.1001` (Pasar Ambacang) | 1 entry |

**Behavior:** Endpoint mengembalikan **array entries** di field `data`, satu
entry per **child level** dari yang diminta. Misalnya:
- `adm1=13` → array 19 kabupaten/kota di Sumbar
- `adm2=13.71` → array 11 kecamatan di Padang
- `adm3=13.71.09` → array 9 kelurahan di Kuranji
- `adm4=13.71.09.1001` → array 1 entry (single kelurahan)

**Contoh URL berfungsi:**

```
https://cuaca.bmkg.go.id/api/df/v1/forecast/adm?adm1=13
https://cuaca.bmkg.go.id/api/df/v1/forecast/adm?adm2=13.71
https://cuaca.bmkg.go.id/api/df/v1/forecast/adm?adm3=13.71.09
https://cuaca.bmkg.go.id/api/df/v1/forecast/adm?adm4=13.71.09.1001
```

<Aside type="caution" title="Endpoint Legacy Hanya Support adm4">
	Endpoint legacy (`api.bmkg.go.id/publik/prakiraan-cuaca`) hanya menerima
	parameter `adm4`. Parameter `adm1`/`adm2`/`adm3` akan return HTML landing
	page, bukan JSON. **Gunakan endpoint v2 (cuaca.bmkg.go.id) untuk semua
	level wilayah.**
</Aside>

## Response Schema

### Top-level Structure

```json
{
  "lokasi": {
    "adm1": "string — nama provinsi",
    "adm2": "string — nama kabupaten/kota",
    "adm3": "string — nama kecamatan",
    "adm4": "string — nama kelurahan",
    "provinsi": "string",
    "kotkab": "string",
    "kecamatan": "string",
    "desa": "string",
    "lon": "number — longitude",
    "lat": "number — latitude",
    "timezone": "string — 'Asia/Jakarta'"
  },
  "data": [
    {
      "lokasi": { /* sama dengan di atas */ },
      "cuaca": [
        [
          /* array of forecasts per interval 3 jam */
        ]
      ]
    }
  ]
}
```

### Per-Forecast Entry

```json
{
  "datetime": "string ISO 8601 UTC — '2026-05-12T16:00:00Z'",
  "utc_datetime": "string — '2026-05-12 16:00:00' (UTC tanpa timezone)",
  "local_datetime": "string — '2026-05-12 23:00:00' (WIB tanpa timezone)",
  "t": "number — suhu °C",
  "tcc": "number — Total Cloud Cover (%)",
  "tp": "number — Total Precipitation (mm)",
  "weather": "number — weather code (lihat tabel di bawah)",
  "weather_desc": "string — deskripsi cuaca (e.g., 'Hujan Ringan')",
  "weather_desc_en": "string — English description",
  "wd_deg": "number — wind direction (derajat)",
  "wd": "string — wind direction origin (e.g., 'NE', 'SW')",
  "wd_to": "string — wind direction destination",
  "ws": "number — wind speed (km/h)",
  "hu": "number — humidity (%)",
  "vs": "number — visibility (m)",
  "vs_text": "string — visibility text (e.g., '< 9 km')",
  "time_index": "string — interval index (e.g., '3-4', '6-7')",
  "analysis_date": "string — '2026-05-12T12:00:00' (waktu BMKG analyze)",
  "image": "string — URL icon SVG cuaca, e.g., 'https://api-apps.bmkg.go.id/storage/icon/cuaca/berawan-pm.svg'"
}
```

<Aside type="tip" title="Field `image` Useful untuk UI">
	Field `image` adalah URL ke icon SVG yang sudah disediakan BMKG (untuk
	day-time dan night-time terpisah). Bisa langsung dipakai di UI tanpa
	perlu mapping weather code → icon sendiri.
</Aside>

## CORS

Endpoint mendukung CORS (`Access-Control-Allow-Origin: *`), sehingga dapat
dipanggil langsung dari aplikasi browser tanpa proxy server.

## Refresh Rate & Caching

- **Data update:** setiap 6 jam (00:00, 06:00, 12:00, 18:00 WIB)
- **Polling lebih cepat tidak menghasilkan data baru** — data sama saja
- **Rekomendasi caching:** cache di sisi server selama 1-2 jam

## Autentikasi

Tidak diperlukan. Endpoint bersifat publik tanpa API key.

## Quick Reference Summary

| Aspek | Value |
|---|---|
| Primary Endpoint | `https://cuaca.bmkg.go.id/api/df/v1/forecast/adm` |
| Legacy Endpoint | `https://api.bmkg.go.id/publik/prakiraan-cuaca` (adm4 only) |
| Method | GET |
| Format | JSON |
| Granularity | adm1 (provinsi) → adm2 (kota) → adm3 (kecamatan) → adm4 (kelurahan) |
| Behavior | Returns array entries (1 per child level) |
| Refresh | 6-12 jam |
| Forecast horizon | **~9 hari** (interval mulai 1 jam di hari awal, melebar ke 3-6 jam di hari akhir) |
| CORS | Enabled |
| Auth | None |
| Required Headers | `Accept: application/json` recommended |
| Rate limit | Tidak dipublikasikan (rekomendasi: cache 1-2 jam) |
