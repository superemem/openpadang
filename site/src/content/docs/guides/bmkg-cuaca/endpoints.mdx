BMKG menyediakan endpoint tunggal untuk prakiraan cuaca dengan parameter
hierarkis sesuai level administratif.

## Base Endpoint

```
GET https://api.bmkg.go.id/publik/prakiraan-cuaca
```

## Query Parameters

<Aside type="caution" title="Penting: Hanya `adm4` yang Work">
	Berdasarkan testing per Mei 2026, **hanya parameter `adm4` (kelurahan)
	yang mengembalikan JSON valid**. Parameter `adm1`, `adm2`, dan `adm3`
	mengembalikan HTML landing page BMKG, bukan data prakiraan.

	Untuk mendapatkan data agregat (kota, kecamatan), Anda perlu melakukan
	multiple request per kelurahan dan menggabungkan di sisi client.
</Aside>

Endpoint menerima parameter level wilayah:

| Parameter | Level | Format | Status | Contoh (Padang) |
|---|---|---|---|---|
| `adm1` | Provinsi | 2 digit | вЭМ Return HTML | `13` (Sumatera Barat) |
| `adm2` | Kabupaten/Kota | 5 digit | вЭМ Return HTML | `13.71` (Kota Padang) |
| `adm3` | Kecamatan | 8 digit | вЭМ Return HTML | `13.71.09` (Kuranji) |
| `adm4` | Kelurahan/Desa | 13 digit | вЬЕ Return JSON | `13.71.09.1001` (Pasar Ambacang) |

**Contoh URL yang berfungsi:**

```
https://api.bmkg.go.id/publik/prakiraan-cuaca?adm4=13.71.09.1001
```

## Response Schema

### Top-level Structure

```json
{
  "lokasi": {
    "adm1": "string вАФ nama provinsi",
    "adm2": "string вАФ nama kabupaten/kota",
    "adm3": "string вАФ nama kecamatan",
    "adm4": "string вАФ nama kelurahan",
    "provinsi": "string",
    "kotkab": "string",
    "kecamatan": "string",
    "desa": "string",
    "lon": "number вАФ longitude",
    "lat": "number вАФ latitude",
    "timezone": "string вАФ 'Asia/Jakarta'"
  },
  "data": [
    {
      "lokasi": { /* sama dengan di atas */ },
      "cuaca": [
        [
          /* array of forecasts per interval 3 jam */
        ]
      ]
    }
  ]
}
```

### Per-Forecast Entry

```json
{
  "datetime": "string ISO 8601 UTC вАФ '2026-05-12T16:00:00Z'",
  "utc_datetime": "string вАФ '2026-05-12 16:00:00' (UTC tanpa timezone)",
  "local_datetime": "string вАФ '2026-05-12 23:00:00' (WIB tanpa timezone)",
  "t": "number вАФ suhu ¬∞C",
  "tcc": "number вАФ Total Cloud Cover (%)",
  "tp": "number вАФ Total Precipitation (mm)",
  "weather": "number вАФ weather code (lihat tabel di bawah)",
  "weather_desc": "string вАФ deskripsi cuaca (e.g., 'Hujan Ringan')",
  "weather_desc_en": "string вАФ English description",
  "wd_deg": "number вАФ wind direction (derajat)",
  "wd": "string вАФ wind direction origin (e.g., 'NE', 'SW')",
  "wd_to": "string вАФ wind direction destination",
  "ws": "number вАФ wind speed (km/h)",
  "hu": "number вАФ humidity (%)",
  "vs": "number вАФ visibility (m)",
  "vs_text": "string вАФ visibility text (e.g., '< 9 km')",
  "time_index": "string вАФ interval index (e.g., '3-4', '6-7')",
  "analysis_date": "string вАФ '2026-05-12T12:00:00' (waktu BMKG analyze)",
  "image": "string вАФ URL icon SVG cuaca, e.g., 'https://api-apps.bmkg.go.id/storage/icon/cuaca/berawan-pm.svg'"
}
```

<Aside type="tip" title="Field `image` Useful untuk UI">
	Field `image` adalah URL ke icon SVG yang sudah disediakan BMKG (untuk
	day-time dan night-time terpisah). Bisa langsung dipakai di UI tanpa
	perlu mapping weather code вЖТ icon sendiri.
</Aside>

## CORS

Endpoint mendukung CORS (`Access-Control-Allow-Origin: *`), sehingga dapat
dipanggil langsung dari aplikasi browser tanpa proxy server.

## Refresh Rate & Caching

- **Data update:** setiap 6 jam (00:00, 06:00, 12:00, 18:00 WIB)
- **Polling lebih cepat tidak menghasilkan data baru** вАФ data sama saja
- **Rekomendasi caching:** cache di sisi server selama 1-2 jam

## Autentikasi

Tidak diperlukan. Endpoint bersifat publik tanpa API key.
