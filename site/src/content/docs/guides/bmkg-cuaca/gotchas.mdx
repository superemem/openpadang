API BMKG Cuaca memiliki sejumlah quirks yang perlu diantisipasi sebelum
deployment. Berikut adalah hasil observasi dari penggunaan production.

## ⚠️ Response Nested Sangat Dalam

Struktur response BMKG Cuaca memiliki nested array 3 level. Banyak kesalahan
parsing muncul dari sini.

```js
// ❌ SALAH: assume struktur flat
const cuaca = data.cuaca;  // undefined

// ✅ BENAR: navigate sesuai struktur
const cuaca = data.data[0].cuaca[0][0];
//                   ^      ^      ^
//                   |      |      └─ index forecast (0 = paling awal)
//                   |      └─ index "day group" (umumnya 0)
//                   └─ index lokasi (umumnya 0)
```

Untuk safety:

```js
const forecasts = data?.data?.[0]?.cuaca?.[0] ?? [];
if (forecasts.length === 0) {
	throw new Error('Forecast data tidak tersedia');
}
```

## ⚠️ Format `datetime` Bukan ISO 8601

Field `datetime` di top level menggunakan format custom: `"202605120700"`
(YYYYMMDDHHMM, **lokal WIB**). BUKAN ISO 8601.

```js
// ❌ SALAH: parse langsung sebagai Date
new Date(forecast.datetime); // Invalid Date

// ✅ BENAR: parse manual atau gunakan field alternatif
const dt = forecast.datetime;
const year = dt.substring(0, 4);
const month = dt.substring(4, 6);
const day = dt.substring(6, 8);
const hour = dt.substring(8, 10);
const min = dt.substring(10, 12);
const local = new Date(`${year}-${month}-${day}T${hour}:${min}:00+07:00`);

// Atau gunakan field yang sudah benar:
const utc = new Date(forecast.utc_datetime); // ISO 8601 UTC
const localISO = new Date(forecast.local_datetime); // ISO 8601 WIB
```

## ⚠️ Field Bertipe Inkonsisten

Tidak semua field numerik bertipe number — beberapa string.

| Field | Tipe Aktual | Catatan |
|---|---|---|
| `t` (suhu) | number | OK |
| `hu` (kelembapan) | number | OK |
| `tp` (presipitasi) | number | OK |
| `ws` (wind speed) | number | OK |
| `weather` | number | Code numerik |
| `time_index` | **string** | "0", "3", "6", dst. Hati-hati parsing. |
| `vs` (visibility) | number (meter) | |
| `vs_text` | string | Human-readable ("10 km") |

```js
// ❌ SALAH: compare time_index sebagai number
if (forecast.time_index > 9) // "12" > 9 → false (string comparison)

// ✅ BENAR: convert dulu
if (parseInt(forecast.time_index) > 9)
```

## ⚠️ Weather Code Tidak Lengkap di Dokumentasi BMKG

BMKG menggunakan subset WMO codes, namun tidak semua kode terdokumentasi
publik. Kode yang umum muncul:

| Code | Description |
|---|---|
| 0, 1, 2 | Cerah / Cerah Berawan |
| 3, 4 | Berawan / Berawan Tebal |
| 5 | Udara Kabur |
| 10, 45 | Asap / Kabut |
| 60 | Hujan Ringan |
| 61 | Hujan Sedang |
| 63 | Hujan Lebat |
| 80 | Hujan Lokal |
| 95, 97 | Hujan Petir |

**Defensive mapping:**

```js
const WEATHER_DESC: Record<number, string> = {
	0: 'Cerah',
	1: 'Cerah Berawan',
	2: 'Cerah Berawan',
	3: 'Berawan',
	4: 'Berawan Tebal',
	5: 'Udara Kabur',
	10: 'Asap',
	45: 'Kabut',
	60: 'Hujan Ringan',
	61: 'Hujan Sedang',
	63: 'Hujan Lebat',
	80: 'Hujan Lokal',
	95: 'Hujan Petir',
	97: 'Hujan Petir',
};

function describeWeather(code: number): string {
	// Fallback ke weather_desc dari BMKG kalau ada
	return WEATHER_DESC[code] ?? 'Tidak diketahui';
}
```

Lebih baik **selalu gunakan field `weather_desc`** yang sudah disediakan BMKG,
fallback ke mapping manual hanya jika kosong.

## ⚠️ Forecast Horizon Variable

Forecast horizon **biasanya 3 hari** dengan interval 3 jam, namun:
- **Tidak selalu** muncul lengkap (kadang hanya 1-2 hari)
- **Time interval bisa tidak konsisten** di akhir series

```js
// Selalu cek jumlah forecast yang tersedia
const forecasts = data.data[0].cuaca[0];
console.log(`Tersedia ${forecasts.length} forecast intervals`);

// Filter forecasts dalam horizon spesifik
const next6Hours = forecasts.filter((f) => {
	const dt = new Date(f.utc_datetime);
	const hourDiff = (dt - Date.now()) / 3_600_000;
	return hourDiff >= 0 && hourDiff <= 6;
});
```

## ⚠️ Refresh Rate 6 Jam — Polling Cepat Sia-Sia

Data BMKG di-update setiap 6 jam (00:00, 06:00, 12:00, 18:00 WIB). Polling
lebih cepat **tidak menghasilkan data baru**, hanya menghabiskan resource.

**Strategi caching yang direkomendasikan:**

```js
// Server-side cache, expire setiap 1 jam
let cache = { data: null, expires: 0 };

export async function GET({ params }) {
	const { adm4 } = params;

	if (cache[adm4]?.expires > Date.now()) {
		return json(cache[adm4].data);
	}

	const res = await fetch(
		`https://api.bmkg.go.id/publik/prakiraan-cuaca?adm4=${adm4}`
	);
	const data = await res.json();

	cache[adm4] = {
		data,
		expires: Date.now() + 60 * 60 * 1000, // 1 jam
	};

	return json(data);
}
```

## ⚠️ Multi-Kelurahan Request — Hati-Hati Burst

Untuk dashboard multi-kelurahan (e.g., 100+ kelurahan Padang), `Promise.all`
yang parallel **dapat memicu rate limit informal**.

```js
// ❌ SALAH: burst 100 request bersamaan
const all = await Promise.all(
	KELURAHAN.map((k) => fetch(`...?adm4=${k.adm4}`))
);

// ✅ BENAR: throttle dengan batch atau concurrency limit
import pLimit from 'p-limit';
const limit = pLimit(5); // max 5 concurrent

const all = await Promise.all(
	KELURAHAN.map((k) =>
		limit(() => fetch(`...?adm4=${k.adm4}`))
	)
);
```

Alternatif: gunakan `adm3` (kecamatan) untuk dapatkan data agregat 11 kecamatan
Padang dengan hanya 11 request.

## ⚠️ Endpoint Kadang Berubah Path

BMKG pernah mengubah path endpoint (dari `api.bmkg.go.id/...` ke variasi
lainnya). Skema yang sama, namun URL berbeda. Selalu monitor.

**Workaround:** wrap endpoint URL sebagai konstanta yang mudah di-update:

```js
const BMKG_CUACA_URL = 'https://api.bmkg.go.id/publik/prakiraan-cuaca';

async function getCuaca(adm: { level: string; code: string }) {
	const url = `${BMKG_CUACA_URL}?${adm.level}=${adm.code}`;
	return fetch(url).then((r) => r.json());
}
```

## ⚠️ Field `lokasi` Duplikat di Top dan Nested

Response memiliki dua representasi lokasi:

```json
{
  "lokasi": { /* top-level */ },
  "data": [
    {
      "lokasi": { /* nested, sama dengan top-level */ }
    }
  ]
}
```

Sebagian besar field identik, namun ada perbedaan minor (e.g., naming).
**Gunakan top-level `lokasi`** untuk konsistensi.

## ⚠️ Outage dan Maintenance

Endpoint BMKG terkadang merespons dengan:
- HTTP 503 (service unavailable)
- HTML error page (bukan JSON)
- Response dengan `data: []` kosong

**Defensive coding:**

```js
try {
	const res = await fetch(BMKG_CUACA_URL + `?adm4=${adm4}`);

	if (!res.ok) throw new Error(`HTTP ${res.status}`);

	const ct = res.headers.get('content-type');
	if (!ct?.includes('json')) throw new Error('Not JSON response');

	const data = await res.json();
	if (!data?.data?.[0]?.cuaca?.[0]?.length) {
		throw new Error('Empty forecast data');
	}

	return data;
} catch (err) {
	console.error('BMKG Cuaca fetch failed:', err);
	return null; // fallback ke cache atau warning
}
```

## Limitations OpenPadang Highlight

| Limitation | Workaround |
|---|---|
| Tidak ada API key/auth | Tidak diperlukan |
| Tidak ada webhook | Polling tiap 1-2 jam |
| Refresh 6 jam | Polling lebih cepat sia-sia, gunakan caching |
| Format datetime non-ISO di top-level | Gunakan `utc_datetime` / `local_datetime` |
| Response nested 3 level | Defensive optional chaining |
| Weather code tidak terdokumentasi penuh | Selalu pakai `weather_desc` BMKG sebagai source |
| Coverage hanya Indonesia | OK untuk use case Padang |
| Multi-kelurahan = banyak request | Throttle, atau pakai `adm3` untuk agregasi |
