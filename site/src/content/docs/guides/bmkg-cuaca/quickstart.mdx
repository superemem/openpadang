<Aside type="tip" title="Setup dalam 5 menit">
	BMKG menyediakan endpoint prakiraan cuaca publik tanpa autentikasi.
	Granularity hingga level kelurahan (adm4) — sangat presisi untuk aplikasi
	hyperlocal weather.
</Aside>

## Apa yang Akan Diperoleh

API BMKG Cuaca menyediakan prakiraan dengan tingkat detail bervariasi:

- **Per kelurahan (adm4)** — granularity terhalus, ideal untuk hyperlocal app
- **Per kecamatan (adm3)** — agregasi tingkat kecamatan
- **Per kabupaten/kota (adm2)** — gambaran umum kota
- **Per provinsi (adm1)** — overview provinsi

Output mencakup prakiraan **3 hari ke depan dengan interval 3 jam**: suhu,
kelembapan, kondisi cuaca, kecepatan angin, dan parameter lainnya.

Seluruh data merupakan **public domain** (data pemerintah) dan dapat digunakan
untuk berbagai keperluan: weather widget, agenda planning, monitoring
agribisnis, hingga sistem warning cuaca ekstrem.

## Langkah-Langkah

<Steps>

1. **Tentukan kode wilayah** menggunakan format BPS standar level kelurahan
   (`adm4`). Contoh: Kelurahan Pasar Ambacang (Kuranji) = `13.71.09.1001`.
   (Lihat referensi lengkap di [Endpoint Reference](/guides/bmkg-cuaca/endpoints/)).

   <Aside type="caution">
     Endpoint **hanya menerima parameter `adm4`** (kelurahan). Parameter
     `adm1`/`adm2`/`adm3` mengembalikan HTML landing page, bukan JSON.
   </Aside>

2. **Akses endpoint** dengan parameter `adm4` dan header `Accept: application/json`:

   ```
   https://api.bmkg.go.id/publik/prakiraan-cuaca?adm4=13.71.09.1001
   ```

   Response berupa data prakiraan dalam format JSON.

3. **Parse response** dalam aplikasi Anda. Struktur response (disederhanakan):

   ```json

       "adm1": "13",
       "adm2": "13.71",

       "timezone": "Asia/Jakarta"
     },
     "data": [
       {
         "lokasi": { "...sama dengan di atas..." },
         "cuaca": [
           [
             {
               "datetime": "2026-05-12T16:00:00Z",
               "utc_datetime": "2026-05-12 16:00:00",
               "local_datetime": "2026-05-12 23:00:00",
   ```

4. **Tampilkan atau proses data** sesuai kebutuhan: dashboard cuaca harian,
   alert hujan, planning event, dan sebagainya.

## Contoh Implementasi

<Tabs>
<TabItem label="cURL">

```bash
curl -s -H "Accept: application/json" -H "User-Agent: curl/8.0" \
  "https://api.bmkg.go.id/publik/prakiraan-cuaca?adm4=13.71.09.1001" \
  | jq '.data[0].cuaca[0][0]'
```

</TabItem>
<TabItem label="JavaScript">

```js
const adm4 = '13.71.09.1001'; // Kelurahan Pasar Ambacang, Kuranji
const res = await fetch(
	`https://api.bmkg.go.id/publik/prakiraan-cuaca?adm4=${adm4}`,
	{ headers: { 'Accept': 'application/json' } }
);
const { lokasi, data } = await res.json();

const cuacaTerkini = data[0].cuaca[0][0];

console.log(`Cuaca ${lokasi.adm4}, ${lokasi.adm3}:`);
console.log(`${cuacaTerkini.weather_desc} · ${cuacaTerkini.t}°C`);
console.log(`Kelembapan ${cuacaTerkini.hu}% · Angin ${cuacaTerkini.ws} km/h`);
```

</TabItem>
<TabItem label="Python">

```python
import requests

adm4 = "13.71.09.1001"  # Kelurahan Pasar Ambacang, Kuranji
url = f"https://api.bmkg.go.id/publik/prakiraan-cuaca?adm4={adm4}"
headers = {"Accept": "application/json"}
res = requests.get(url, headers=headers).json()

lokasi = res["lokasi"]
cuaca = res["data"][0]["cuaca"][0][0]

print(f"Cuaca {lokasi['adm4']}, {lokasi['adm3']}:")
print(f"{cuaca['weather_desc']} · {cuaca['t']}°C")
print(f"Kelembapan {cuaca['hu']}% · Angin {cuaca['ws']} km/h")
```

</TabItem>
<TabItem label="SvelteKit">

```ts
// src/routes/api/cuaca/[adm4]/+server.ts
import { json } from '@sveltejs/kit';

export async function GET({ params }) {
	const { adm4 } = params;
	const res = await fetch(
		`https://api.bmkg.go.id/publik/prakiraan-cuaca?adm4=${adm4}`,

	return json(data);
```

## Cuaca untuk Multi-Kelurahan Padang

Padang memiliki 11 kecamatan dengan 104 kelurahan. Untuk dashboard yang
menampilkan multiple lokasi, lakukan parallel request:

```js
const PADANG_LOKASI = [
	{ name: 'Padang Selatan', adm4: '13.71.01.1001' },
	{ name: 'Padang Timur', adm4: '13.71.02.1001' },
	{ name: 'Padang Barat', adm4: '13.71.03.1001' },
	{ name: 'Padang Utara', adm4: '13.71.04.1001' },
	{ name: 'Kuranji (Pasar Ambacang)', adm4: '13.71.09.1001' },
	// ... tambahkan kelurahan lain sesuai kebutuhan
];

const results = await Promise.all(
	PADANG_LOKASI.map(async (lokasi) => {
		const res = await fetch(
			`https://api.bmkg.go.id/publik/prakiraan-cuaca?adm4=${lokasi.adm4}`
		);
		const data = await res.json();
		return { ...lokasi, cuaca: data.data[0].cuaca[0][0] };
	})
);

console.table(
	results.map((r) => ({
		Lokasi: r.name,
		Cuaca: r.cuaca.weather_desc,
		Suhu: `${r.cuaca.t}°C`,
		Kelembapan: `${r.cuaca.hu}%`,
	}))
);
```

<Aside type="caution">
	BMKG tidak mempublikasikan rate limit eksplisit. Untuk multi-request,
	rekomendasi: gunakan caching di sisi server dan polling tidak lebih cepat
	dari 1 jam (data refresh setiap 6 jam).
</Aside>
