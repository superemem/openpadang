	BMKG menyediakan endpoint prakiraan cuaca publik tanpa autentikasi.
	Granularity hingga level kelurahan (adm4) — sangat presisi untuk aplikasi
	hyperlocal weather.
</Aside>

## Apa yang Akan Diperoleh

API BMKG Cuaca menyediakan prakiraan dengan tingkat detail bervariasi:

- **Per kelurahan (adm4)** — granularity terhalus, ideal untuk hyperlocal app
- **Per kecamatan (adm3)** — agregasi tingkat kecamatan
- **Per kabupaten/kota (adm2)** — gambaran umum kota
- **Per provinsi (adm1)** — overview provinsi

Output mencakup prakiraan **3 hari ke depan dengan interval 3 jam**: suhu,
kelembapan, kondisi cuaca, kecepatan angin, dan parameter lainnya.

Seluruh data merupakan **public domain** (data pemerintah) dan dapat digunakan
untuk berbagai keperluan: weather widget, agenda planning, monitoring
agribisnis, hingga sistem warning cuaca ekstrem.

## Langkah-Langkah

<Steps>

1. **Tentukan kode wilayah** menggunakan format BPS standar level kelurahan
   (`adm4`). Contoh: Kelurahan Pasar Ambacang (Kuranji) = `13.71.09.1001`.
   (Lihat referensi lengkap di [Endpoint Reference](/guides/bmkg-cuaca/endpoints/)).

   <Aside type="tip" title="Endpoint v2 untuk Semua Level">
     Endpoint v2 (`cuaca.bmkg.go.id/api/df/v1/forecast/adm`) mendukung
     adm1 hingga adm4. Endpoint legacy (`api.bmkg.go.id/publik/prakiraan-cuaca`)
     hanya mendukung adm4.
     hanya mendukung adm4.
   </Aside>

2. **Akses endpoint v2** dengan parameter level wilayah dan header `Accept: application/json`:

   ```
   https://cuaca.bmkg.go.id/api/df/v1/forecast/adm?adm4=13.71.09.1001
   ```

   Atau untuk level lebih tinggi:

   ```
   https://cuaca.bmkg.go.id/api/df/v1/forecast/adm?adm3=13.71.09   # kecamatan
   https://cuaca.bmkg.go.id/api/df/v1/forecast/adm?adm2=13.71      # kota
   https://cuaca.bmkg.go.id/api/df/v1/forecast/adm?adm1=13         # provinsi
   ```

   Response berupa data prakiraan dalam format JSON, dengan jumlah entries
   sesuai dengan child level (e.g., adm3 returns array kelurahan).

3. **Parse response** dalam aplikasi Anda. Struktur response (disederhanakan):


## Contoh Implementasi

<Tabs>
<TabItem label="cURL">

```bash
# Single kelurahan
curl -s -H "Accept: application/json" \
  "https://cuaca.bmkg.go.id/api/df/v1/forecast/adm?adm4=13.71.09.1001" \
  | jq '.data[0].cuaca[0][0]'

# Semua kelurahan di Kuranji (1 request, 9 kelurahan)
curl -s -H "Accept: application/json" \
  "https://cuaca.bmkg.go.id/api/df/v1/forecast/adm?adm3=13.71.09" \
  | jq '.data[].lokasi.desa'
```

</TabItem>
<TabItem label="JavaScript">

```js
const adm4 = '13.71.09.1001'; // Kelurahan Pasar Ambacang, Kuranji
const res = await fetch(
	`https://cuaca.bmkg.go.id/api/df/v1/forecast/adm?adm4=${adm4}`,
	{ headers: { 'Accept': 'application/json' } }
);
const { lokasi, data } = await res.json();

const cuacaTerkini = data[0].cuaca[0][0];

console.log(`Cuaca ${lokasi.adm4}, ${lokasi.adm3}:`);
console.log(`${cuacaTerkini.weather_desc} · ${cuacaTerkini.t}°C`);
console.log(`Kelembapan ${cuacaTerkini.hu}% · Angin ${cuacaTerkini.ws} km/h`);
```

</TabItem>
<TabItem label="Python">

```python
import requests

adm4 = "13.71.09.1001"  # Kelurahan Pasar Ambacang, Kuranji
url = f"https://cuaca.bmkg.go.id/api/df/v1/forecast/adm?adm4={adm4}"
headers = {"Accept": "application/json"}
res = requests.get(url, headers=headers).json()

lokasi = res["lokasi"]

print(f"Cuaca {lokasi['adm4']}, {lokasi['adm3']}:")
print(f"{cuaca['weather_desc']} · {cuaca['t']}°C")
print(f"Kelembapan {cuaca['hu']}% · Angin {cuaca['ws']} km/h")
```

## Cuaca untuk Multi-Kelurahan Padang

Untuk mendapatkan data semua kelurahan di satu kecamatan, cukup **1 request
dengan parameter `adm3`** — endpoint v2 mengembalikan array semua kelurahan
sebagai child level:

```js
// Single request → semua 9 kelurahan di Kecamatan Kuranji
const res = await fetch(
	'https://cuaca.bmkg.go.id/api/df/v1/forecast/adm?adm3=13.71.09',
	{ headers: { 'Accept': 'application/json' } }
);
const { lokasi, data } = await res.json();

console.log(`Kecamatan: ${lokasi.kecamatan}`);
console.log(`Jumlah kelurahan: ${data.length}`);

data.forEach((entry) => {
	const cuaca = entry.cuaca[0][0]; // forecast terkini
	console.log(`  ${entry.lokasi.desa}: ${cuaca.weather_desc} · ${cuaca.t}°C`);
});
```

Untuk **semua 11 kecamatan Padang sekaligus**, gunakan `adm2=13.71`:

```js
const res = await fetch(
	'https://cuaca.bmkg.go.id/api/df/v1/forecast/adm?adm2=13.71',
	{ headers: { 'Accept': 'application/json' } }
);
const { lokasi, data } = await res.json();
// data = array 11 kecamatan, masing-masing punya cuaca forecast
```

<Aside type="tip">
	Endpoint v2 menghemat banyak request. Sebelumnya butuh 11 request untuk
	semua kecamatan Padang, sekarang cukup 1 request dengan `adm2=13.71`.
</Aside>

<Aside type="caution">
	BMKG tidak mempublikasikan rate limit eksplisit. Tetap gunakan caching di
	sisi server, dengan TTL minimal 1 jam (data refresh setiap 6 jam).
</Aside>
